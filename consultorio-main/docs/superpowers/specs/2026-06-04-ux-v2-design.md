# UX/UI v2 — Design Spec

**Data:** 2026-06-04  
**Status:** Aprovado para implementação

---

## 1. Visão Geral

Melhoria da experiência do usuário do sistema de consultório médico em 5 áreas:

1. **Tradução para espanhol** — toda a interface passa de português para espanhol
2. **Excluir missionário** — exclusão permanente com diálogo de confirmação
3. **Seletor de data moderno** — substitui inputs nativos por um calendário visual (`react-day-picker`)
4. **Agenda semanal** — tira de dias clicáveis na tela de agenda
5. **Saudação personalizada** — "Dr. Quezada" lido dos metadados do usuário no Supabase

## 3. Excluir Missionário

### 3.1 Onde aparece o botão

- Página de perfil do missionário (`app/missionaries/[id]/page.tsx`) — botão "Eliminar" no cabeçalho, ao lado de "Editar"

### 3.2 Comportamento

1. Usuário clica em "Eliminar"
2. Modal de confirmação abre com:
   - Ícone ⚠️
   - Título: "¿Eliminar misionero?"
   - Corpo: "Se eliminarán permanentemente **[preferred_name]** y todas sus consultas y citas. Esta acción no se puede deshacer."
   - Botão "Cancelar" (cinza)
   - Botão "Sí, eliminar" (vermelho)
3. Ao confirmar: chama server action `deleteMissionary(id)`
4. Redireciona para `/missionaries`

### 3.3 Server action

Arquivo: `lib/actions/missionaries.ts` — nova função `deleteMissionary`:
```typescript
export async function deleteMissionary(id: string) {
  // DELETE cascata via FK — apaga consultations e appointments automaticamente
  // Redireciona para /missionaries
}
```

### 3.4 Componente

Novo componente cliente: `components/missionaries/delete-missionary-button.tsx`
- Gerencia estado `open` do modal de confirmação
- Chama `deleteMissionary` via server action

## 4. Seletor de Data Moderno

### 4.1 Biblioteca

`react-day-picker` v9 — 20KB, sem dependências, suporte a locale espanhol (`es` do `date-fns`).

### 4.2 Novo componente

`components/ui/date-picker.tsx` — componente reutilizável:
- Props: `value: Date | undefined`, `onChange: (date: Date) => void`, `label: string`, `required?: boolean`
- Exibe campo com texto formatado (ex: "Miér 4 de junio, 2025")
- Abre calendário inline ao clicar
- Fecha ao selecionar

`components/ui/datetime-picker.tsx` — para campos com data + hora:
- Mesmas props + `showTime: true`
- Calendário + seletores de hora/minuto

### 4.3 Onde substituir

| Arquivo | Campo | Tipo |
|---|---|---|
| missionary-form.tsx | birthdate, mission_start_date, mission_expected_end | DatePicker |
| consultation-form.tsx | consulted_at, follow_up_date | DateTimePicker / DatePicker |
| appointment-form.tsx | scheduled_at | DateTimePicker |

### 4.4 Integração com formulários

Os componentes gerenciam estado interno (`useState`) e escrevem o valor em um `<input type="hidden">` com o mesmo `name` para compatibilidade com os Server Actions existentes (que já leem via `formData.get()`).

## 5. Agenda Semanal

### 5.1 Novo componente

`components/agenda/week-strip.tsx`:
- Mostra os 7 dias da semana atual, começando na **segunda-feira** (L M M J V S D)
- Dia selecionado: círculo violeta
- Dias com consultas: ponto abaixo do número
- Clique em um dia: navega para `/agenda?date=YYYY-MM-DD`
- Setas ‹ / › para semana anterior/próxima
- Mês/ano no topo

### 5.2 Integração na agenda

`app/agenda/page.tsx`:
- Substitui a navegação "Anterior / Próximo" por `WeekStrip`
- `WeekStrip` recebe `selectedDate` e `datesWithAppointments` (array de strings YYYY-MM-DD)
- `getAppointmentDatesInMonth` já existe em `lib/queries/agenda.ts` — usado para calcular os pontos

## 6. Saudação Personalizada

### 6.1 Fonte dos dados

O nome do médico é lido de `user.user_metadata.display_name` via `supabase.auth.getUser()` no server component do dashboard.

### 6.2 Setup necessário

Após deploy, o usuário deve atualizar o metadado via Supabase dashboard:
- Authentication → Users → [email do médico] → Edit → User Metadata:
```json
{ "display_name": "Dr. Quezada" }
```

### 6.3 Fallback

Se `display_name` não estiver definido, mostra apenas a saudação sem nome: "Buenos días 👋"

### 6.4 Dashboard

`app/dashboard/page.tsx`:
- Busca `user` via `supabase.auth.getUser()` (já está no contexto de server component)
- Extrai `user.user_metadata?.display_name ?? ''`
- Saudação: `Buenos días, ${displayName} 👋` (ou sem nome se vazio)
- Data em espanhol via `toLocaleDateString('es-BO', ...)`
